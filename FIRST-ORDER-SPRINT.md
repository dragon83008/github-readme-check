# 30 分钟第一单冲刺

目标：今天尽快拿到第一笔 10 元，而不是先把所有东西做完美。

## 0 到 5 分钟：准备成交入口

先把 `index.html` 里的联系方式换成你的：

- 微信号
- 邮箱，没邮箱就删掉那一行
- 收款方式说明

如果暂时还没发布 GitHub Pages，也可以先直接发文字接单。

## 5 到 15 分钟：找 20 个可能需要的人

优先找这几类人：

- 最近在找实习的人
- 简历里放了 GitHub 的人
- 正在做毕业设计或课程项目的人
- 发过“求简历建议”“求项目建议”的同学
- 写了开源项目但 README 很短的人

不要群发广告。先发给认识的人，成交概率更高。

## 15 到 25 分钟：逐个私信

模板 1，适合熟人：

```text
我今天在试一个 10 元小服务：帮人看 GitHub 主页或项目 README，给 3 条最该先改的建议，再写一版能直接复制的 README 开头。

你要不要让我看一下？今天能给你。
```

模板 2，适合半熟人：

```text
看到你有 GitHub 项目，我现在做一个 10 元 README 快速体检。

交付很简单：3 条问题 + 一版可复制开头 + 一个下一步建议。适合放简历或作品集前先改一下第一印象。

要不要试一次？你把链接发我就行。
```

模板 3，适合群里发：

```text
我做了一个免费的 GitHub README 自测页，也顺便接 3 个 10 元人工体检名额。

适合找实习、改简历项目、做作品集的人。你可以先自己测分；如果想让我人工看，我给你 3 条最该先改的建议和一版可复制 README 开头。

想试的私信我链接。
```

## 25 到 30 分钟：成交

有人回复“多少钱 / 怎么弄 / 可以”时，直接发：

```text
10 元。你把 GitHub 链接发我，付款后我今天给你一份简短诊断：

1. 3 个最该先改的问题
2. 一版 README 开头
3. 一个下一步建议

改完后可以免费追问 1 次。
```

## 没人付款怎么办

不要立刻换项目，先做一个免费样本。

找一个朋友的 GitHub，免费写一份 5 行诊断，然后问：

```text
我能不能把这个作为示例发出来？会隐藏你的个人信息。
```

拿到同意后，把示例发到朋友圈或群里，再附一句：

```text
今天继续接 3 个，10 元一个。
```

## 收到第一笔 10 元后

马上做这 3 件事：

1. 在 `product/order-tracker.csv` 记录订单。
2. 用 `product/delivery-template.md` 交付。
3. 把页面价格先保留 10 元，再接 2 单后改成 19 元。
