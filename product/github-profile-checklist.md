# GitHub 主页 / README 快速检查清单

## 主页第一屏

- 是否能在 5 秒内看懂你是谁？
- 是否写清楚你正在做什么方向？
- 是否有 1 到 3 个最值得看的项目？
- 是否能看到联系方式或下一步动作？

## 项目 README

- 项目名是否具体？
- 第一段是否说明“解决什么问题”？
- 是否有截图、演示链接或运行结果？
- 是否有安装和使用步骤？
- 是否说明技术栈？
- 是否说明项目亮点？
- 是否把未完成事项写清楚？

## 求职向优化

- 把最像真实产品的项目放前面。
- 删掉空仓库、重复仓库、明显半成品。
- README 第一段少写“学习中”，多写“我做了什么”。
- 项目描述要有动词，例如“实现、整理、封装、分析、自动化、可视化”。

## 接单向优化

- 写清楚你能帮别人完成什么。
- 放 2 到 3 个可交付案例。
- 给出联系方式。
- 给出服务范围和不接的范围。

## 开源向优化

- 写清楚安装方式。
- 写清楚贡献方式。
- 标注许可证。
- 给 issue 模板和最小复现要求。
